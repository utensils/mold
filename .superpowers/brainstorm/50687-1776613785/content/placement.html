Where does the generate UI live?

The biggest shape-defining choice. Pick the one that fits how you'll use it. (Click to select — you can change later.)

A

Inline prompt bar on gallery

A persistent rounded prompt input pinned above the feed — like ChatGPT's composer or Midjourney Web. Advanced knobs collapse into a small "▼ options" pill next to it. Results stream in at the top of the feed.

Feels like: Twitter composer. Generation is just "another thing the gallery does."
Trade-off: Power knobs are one click away, not visible. One prompt in flight at a time.

B

Slide-over drawer from the right

A "✨ Generate" button in the TopBar opens a right-side drawer (mirrors the existing DetailDrawer pattern). Full form lives in the drawer — prompt, model, size, advanced knobs. Submit streams progress, result appears in the gallery beneath.

Feels like: A focused panel, doesn't steal real estate from browsing.
Trade-off: An extra click to start typing. Mobile would go fullscreen (already the drawer pattern).

C

Dedicated /generate route

A two-tab top-level switch: Gallery / Generate. Generate gets its own page — form on the left, live preview/queue on the right.

Feels like: A1111 / ComfyUI / "it's a studio app now."
Trade-off: Requires a router (currently zero-router SPA). Divorces "make" from "see what I've made."

D

Hybrid: inline bar + expand-to-drawer

Always-visible compact prompt + model + go button at the top of the gallery. A "⚙ advanced" icon expands into the slide-over drawer for steps/guidance/seed/lora/source-image. Default flow stays one line.

Feels like: Best of A + B. Most users never open the drawer.
Trade-off: Two code paths. Slightly more to build, but no dead-end states.
